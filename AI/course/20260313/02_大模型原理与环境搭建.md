## 第一部分：大模型基本原理（通俗版）（1.5 小时）

### 一、从人脑到 AI 脑

**人脑如何处理语言：**
```
听到"今天天气真好"
  ↓
大脑分析：今天（时间）+ 天气（主题）+ 真好（评价）
  ↓
理解：说话人心情不错，可能在建议出门
  ↓
回应："是啊，适合出去走走"
```

**AI 如何处理语言：**
```
输入"今天天气真好"
  ↓
分词：["今天", "天气", "真", "好"]
  ↓
转向量：[0.23, -0.45, 0.67, ...]（每个词变成数字列表）
  ↓
Transformer 处理：计算词与词之间的关系
  ↓
输出预测：下一个词最可能是"啊"、"呀"、"，"等
```

### 二、Transformer 核心思想

**问题：传统模型为什么慢？**

```
RNN 模型（旧方法）：
输入：我 爱 你
      ↓
    "我" → 处理 → 传给下一步
              ↓
            "爱" → 处理 → 传给下一步
                    ↓
                  "你" → 处理 → 输出

缺点：必须按顺序，无法并行，慢！
```

**Transformer 的解决方案：**

```
Transformer（新方法）：
输入：我 爱 你
      ↓   ↓   ↓
    同时处理所有词！
      ↓   ↓   ↓
    通过"注意力"机制理解关系
      ↓
    输出

优点：可以并行，快！
```

### 三、自注意力机制（Self-Attention）

**核心公式（不用背，理解意思就行）：**
```
Attention(Q, K, V) = softmax(QK^T / √d) × V
```

**通俗解释：**
```
想象你在一个聚会上：

Q（Query）= 你想找什么样的人聊天
K（Key）= 每个人身上的标签（"程序员"、"设计师"等）
V（Value）= 每个人实际能聊的内容

Attention 的过程：
1. 你扫视全场（QK^T 计算匹配度）
2. 找到标签匹配的人（softmax 归一化）
3. 过去跟他们聊天（乘以 V 获取信息）
```

**实际例子：**
```
句子："银行发放了贷款"

处理"贷款"这个词时，模型会关注：
- "银行"（谁发放的？）→ 高注意力
- "发放"（什么动作？）→ 高注意力
- "了"（时态）→ 低注意力

这样模型就理解了："银行"是主体，"发放"是动作，"贷款"是对象
```

### 四、大模型的"思考"过程

**步骤分解：**

```
第 1 步：输入处理
"今天天气不错" → 分词 → ["今天", "天气", "不错"]

第 2 步：词嵌入
每个词转成向量（512 维或 768 维的数字列表）
"今天" → [0.23, -0.45, 0.67, ..., 0.12]

第 3 步：位置编码
加上位置信息，让模型知道词的顺序
"今天"（位置 0）+ 位置编码 → 新的向量

第 4 步：多层 Transformer
通过 12 层、24 层或更多层处理
每层都做自注意力计算

第 5 步：输出预测
计算下一个词的概率分布
"今天天气不错，" → 下一个词可能是：
  - "适合" (30%)
  - "我们" (25%)
  - "出去" (20%)
  - ...

第 6 步：采样输出
根据概率选一个词，重复上述过程
```

### 五、关键概念速查表

| 术语 | 英文 | 通俗解释 | 例子 |
|------|------|---------|------|
| Token | Token | 文本的基本单位 | "hello"=1 个 token，"你好"=2 个 tokens |
| 上下文窗口 | Context Window | 模型一次能"记住"多长的内容 | GPT-4: 128K tokens ≈ 10 万字 |
| 温度 | Temperature | 控制输出的随机性 | 0=最确定，1=最随机 |
| Prompt | Prompt | 给模型的指令/问题 | "请帮我写一个 Python 函数" |
| 微调 | Fine-tuning | 在特定数据上继续训练 | 用银行客服对话微调模型 |
| RAG | Retrieval-Augmented Generation | 检索 + 生成 | 先查知识库，再让 AI 回答 |
| Agent | Agent | 能自主完成任务的 AI | "帮我订机票"→AI 自动完成 |

## 第二部分：开发环境搭建（手把手）（2 小时）

### 准备工作

**系统要求：**
- 操作系统：Windows 10+/macOS 10.15+/Linux
- 内存：8GB 以上（推荐 16GB）
- 硬盘：至少 10GB 可用空间
- 网络：能访问外网（部分资源需要）

### 步骤 1：安装 Python

**检查是否已安装：**
```bash
# Mac/Linux
python3 --version

# Windows
python --version
```

**如果未安装：**

**Windows 用户：**
1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.10.x（推荐 3.10.11）
3. 运行安装程序
4. ⚠️ 重要：勾选"Add Python to PATH"
5. 点击"Install Now"

**Mac 用户：**
```bash
# 方法 1：官网下载
# 访问 https://www.python.org/downloads/mac-osx/

# 方法 2：用 Homebrew（推荐）
brew install python@3.10
```

**Linux 用户：**
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.10 python3.10-venv python3-pip

# CentOS/RHEL
sudo yum install python3.10
```

**验证安装：**
```bash
python3 --version
# 应该输出：Python 3.10.x
```

### 步骤 2：安装 Anaconda（推荐）

**为什么用 Anaconda？**
- 一键安装 Python + 常用科学计算库
- 方便管理多个虚拟环境
- 避免各种依赖问题

**安装步骤：**

**Windows：**
1. 访问 https://www.anaconda.com/download
2. 下载 Windows 版本（64-Bit）
3. 运行安装程序
4. 按默认选项安装即可

**Mac：**
```bash
# 方法 1：官网下载
# 访问 https://www.anaconda.com/download

# 方法 2：用 Homebrew
brew install --cask anaconda
```

**Linux：**
```bash
# 下载安装脚本
wget https://repo.anaconda.com/archive/Anaconda3-2023.09-0-Linux-x86_64.sh

# 运行安装
bash Anaconda3-2023.09-0-Linux-x86_64.sh

# 按提示操作，最后重启终端
```

**验证安装：**
```bash
conda --version
# 应该输出版本号，如 conda 23.9.0
```

### 步骤 3：创建虚拟环境

**什么是虚拟环境？**
> 想象成一个独立的"小房间"，每个项目有自己的 Python 和库，互不干扰

**创建环境：**
```bash
# 创建名为 ai-learning 的环境，Python 版本 3.10
conda create -n ai-learning python=3.10

# 确认安装
# 输入 y 确认
```

**激活环境：**
```bash
# Mac/Linux
conda activate ai-learning

# Windows
conda activate ai-learning
```

**激活后，命令行前面会有环境名：**
```
(ai-learning) $  ← 表示已在 ai-learning 环境中
```

**退出环境：**
```bash
conda deactivate
```

### 步骤 4：安装基础库

**常见问题：**


**问题 3：依赖冲突**


**A：**
```bash
# 使用国内镜像
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple <包名>
```

### Q4：API Key 安全吗？会泄露吗？

**A：**
- 不要把 API Key 直接写在代码里上传 GitHub
- 用环境变量或配置文件存储
- 定期更换 API Key

**正确做法：**
```python
import os
api_key = os.getenv("DASHSCOPE_API_KEY")  # 从环境变量读取
```
